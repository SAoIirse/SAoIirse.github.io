BASE家族

base64

base64加密原理：

base64加密本质是一种编码形式，数据以一种二进制文本的形式传递，编码过程分为三步
1.将ASCII转换为对应的八位二进制数据
2.合并起来，再以6位一组重新分配
3.六位值映射到base64字符表
位数不足时，在末尾补0，直至补齐组内位数

特征识别：

● 字符串编码表识别
● 加密填充符识别（一般为=号）
● bit：3 * 8变4 * 6
● 输入参数会被移位拼接，移位位数为 2、4、6位，将3字节拆成4字节
● 理解编码原理，编码时通常都会用3个字节一组来处理比特位数据，这些特征都可以用来分析识别。
常规魔改：编码表（TLS、SMC等各种反调试位置）

RC4

RC4算法的核心思想是利用伪随机数生成器和密钥共同生成一个密钥流，使用该密钥流与明文进行异或运算得到密文。

加密原理

核心加密原理就是和密钥流异或，RC4是一个对称性的加密方式。因此：
加密时：明文^密钥
解密时：密文^密钥

密钥流形成：
1.初始化S盒：创建一个长度为256字节的数组S【256】。
2.创建临时数组 T：创建一个同样长度为256的临时数组 T。
3.初始置换 S：使用一个变量 j = 0，遍历 i 从 0 到 255，对数组 S 进行初步打乱。

伪随机生成:
1.初始化索引：将两个索引指针 i 和 j 都重置为 0。
2.为每一个待处理的数据字节生成密钥字节：对于明文或密文中的每一个字节，循环执行以下步骤：
更新 i：i = (i + 1) % 256
更新 j：j = (j + S[i]) % 256
交换：swap(S[i], S[j])
注意，状态数组S在整个加密/解密过程中是不断动态变化的
生成密钥字节 t：t = (S[i] + S[j]) % 256

输出密钥字节 K：K = S[t]

特征识别：

最明显的是带有256取余和256循环
含有密钥，且带有异或计算
带有伪随机数生成的特征

TEA

一种分组加密算法

加密原理

TEA算法使用64位的明文分组和128位的密钥，使用Feistel分组加密框架，需要进行 64 轮迭代。算法使用了一个常数δ作为倍数，δ来源于黄金比率，以保证每一轮加密都不相同。但δ的精确值并不重要，这里 TEA 把它定义为 δ=「(√5 - 1)231」（也就是程序中的 0×9E3779B9）。之后由于TEA算法被发现含有缺陷，于是衍生出了XTEA和XXTEA，算法是相同的，但对密钥的操作顺序不太相同，增强了安全性。

特征识别

tea算法的主要特征表现在sum和delta变量，以及3行核心加密中出现的右移4左移5，两行各有3个小括号互相异或

AES

加密原理

1.AES是对称加密，密钥是AES算法实现的根本，特别的是该算法中加密解密的密钥是同一个
AES支持三种长度的密钥：
128位，192位，256位
所以我们常见的AES128，AES192，AES256，实际上就是指的AES算法对不同长度密钥的使用。
2.AES还有分组加密特性，也就是在加密时将明文，拆分成一个一个独立的明文块，每个明文长度为128bit.这些明文块经过AES加密器的复杂处理，生成一个个独立的密文块，这些密文块拼接在一起，就是最终的AES加密结果。

特征识别

1.16位字符串，并且字符串拆分成字节的形式存储
2.S盒替换（S_BOX[v31]）、行位移（<<8）
3.常数异或
