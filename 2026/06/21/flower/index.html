逆向花指令

花指令：既在代码中添加永恒跳转的指令，或者垃圾数据

目的是干扰ida等软件对程序的静态分析。使这些软件无法正常反汇编出原始代码。

花指令的实现方式:核心在于制造反汇编歧义
常见手法包括：
1）无效跳转和垃圾字节；
2）利用反汇编线性扫描缺陷；
3）修改返回地址；
4）多态变形

俗话说，知己知彼，百战不殆。花指令为了干扰软件的静态分析，当然就要了解ida常用的反汇编算法，这样才能想办法绕过检测。

ida常用的反汇编算法：
1.线性扫描算法：逐行反汇编（无法将数据和内容进行区分）
2.递归行进算法：按照代码可能的执行顺序进行反汇编程序

在程序中会执行的花指令

可执行花指令指的是花指令代码在程序的正常执行过程中会被执行，但执行这些代码没有任何意义，执行前后不改变任何寄存器的值，同时这部分代码也会被反汇编器正常识别

压栈后恢复栈地址

通过压栈和修改栈指针来干扰反汇编器，但实际执行时会恢复栈指针，使得代码能正常执行。通常用于制造反汇编错误，使得静态分析工具无法正确识别指令边界，隐藏真实的代码逻辑。
常见：
在汇编语言中push是入栈，pop是出栈
使用push指令将一些数据（可能是垃圾数据或有用数据）压入栈中。然后使用pop指令或者直接修改ESP（SP：储存栈顶地址）来恢复栈指针，使得栈平衡。

  1. 将eax压栈（注意，这里eax的值可能是任意的，也可能是后面要用的）
  2. 跳转到label，然后弹出栈顶到eax，这样eax的值就恢复了（如果之前是有效值）或者变成了之前压入的值（如果是垃圾数据，那么eax就被改变了，在压栈出栈后会去除这些垃圾数据

在call指令中嵌入垃圾字节，使得反汇编器从call的目标地址开始错误解析，然后通过ret指令跳转到正确的地址继续执行。

混淆特征码

在保持程序原始功能的基础上改变代码的字节序列，使得基于静态特征码的检测失效。
插入无效指令：如nop、push/pop同一寄存器、加减零等。
指令替换：用功能相同的其他指令替换原指令。
指令重排：改变指令顺序，通过跳转连接。
就是类似于一个能够顺序执行的程序，变成花指令后将其中一些指令变成子程序，通过call，jmp等跳转指令连接
代码变形：将一段代码转换为另一段功能相同但字节表示不同的代码。

  2. 在ret处下断点，查看返回地址
  3. dump内存中的解密代码

在程序中不执行的花指令

简单的E8跳转

在汇编语言中，我们知道call指令的作用就是调用子程序
E8 xx xx xx xx ; call 相对地址偏移（E8是call对应的机器码）
机器码结构：E8 + 4字节相对偏移
作用：调用子程序，将返回地址(push eip)压栈，然后跳转

jz和jnz条件跳转

通过设置标志位，使得条件跳转的条件永远不满足（或永远满足），从而控制执行流不会进入花指令块。（zf标志寄存器第6位，记录其结果是否为0）
● jz / je：当零标志（ZF）为1时跳转
● jnz / jne：当零标志（ZF）为0时跳转

我们来看一个花指令执行前后程序的对比。

在实际程序中：

eax也是寄存器，将eax寄存器的值和自己异或，结果是eax=0
xor是异或运算：
运算规则是，两个位相同则结果为0，不同则为1
为什么ZF=1？
ZF运算规则是：最近一次的运算结果是0，ZF被设置为1；最近一次运算结果非0，ZF被设置为0

在反汇编器中：

永真跳转

永真跳转就是构造一个总是会发生的跳转，让程序执行流必然跳过一段代码，这段被跳过的代码就是不会执行的花指令。
这部分不会被执行的指令，就是用来迷惑反汇编器，让它误以为这部分指令会执行，从而分析错误.

无条件跳转

标志位设置+条件跳转：

永远成立的条件：

test eax, eax 将eax与自身进行按位与操作，结果与eax本身相同（因为任何数和自身按位与还是本身）。然后根据结果设置标志位，其中最重要的是：
● ZF（零标志）：如果结果为0，则ZF=1；否则ZF=0。
● SF（符号标志）：如果结果的最高位为1，则SF=1；否则SF=0。
● PF（奇偶标志）：根据结果中1的个数，如果1的个数为偶数则PF=1，奇数则PF=0。
● CF（进位标志）和OF（溢出标志）：test指令会将CF和OF清零。

常见的脏字节

call immed16 —-> E8 // 3字节指令，immed16为2字节，代表跳转指令的目的地址与下一条指令地址的距离
call immed32 —-> 9A // 5字节指令，immed32为4字节，代表跳转指令的目的地址与下一条指令地址的距离
jmp immed8 —-> EB
jmp immed16 —-> E9
jmp immed32 —-> EA
loop immed8 —-> E2
jz 74 为0则跳转–>短跳转
—->0F 84（近跳转）
jnz 75（短跳转）、0F 85（近跳转）
ret —-> C2
retn —-> C3
